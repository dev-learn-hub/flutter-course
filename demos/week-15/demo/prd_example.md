# 产品需求文档示例（PRD）

> 第15周案例：简易 PRD 结构 + 用户故事示例，供课堂与作业参考。完整模板见课程大纲 week-15。

---

## 1. 文档概述

- **产品名称**：待办清单 App（示例）
- **文档版本**：0.1
- **最后更新**：与课程同步
- **说明**：用于演示 PRD 与用户故事的写法，非完整产品规格。

---

## 2. 产品概述

### 2.1 产品定位

- **目标用户**：需要管理日常任务的学生与职场人。
- **用户痛点**：任务易忘、优先级不清、多设备不同步。
- **价值主张**：简单、快速的待办记录与完成追踪。

### 2.2 核心功能（示例）

1. 待办的新增、编辑、删除
2. 待办状态：未完成 / 已完成
3. 列表展示与简单筛选
4. 本地持久化（如 SharedPreferences 或文件）

---

## 3. 用户故事示例

### 故事 1：添加待办

**作为** 学生用户  
**我想要** 快速添加一条待办并设置标题  
**以便** 不会忘记要做的事  

**接受标准：**

- [ ] 提供输入框与「添加」按钮
- [ ] 添加后列表立即显示新项
- [ ] 标题为空时给出提示，不添加空项

**优先级**：高 | **估算**：2 小时 | **分配给**：—

### 故事 2：标记完成

**作为** 学生用户  
**我想要** 将待办标记为已完成  
**以便** 区分未做与已做  

**接受标准：**

- [ ] 列表项可切换「未完成 / 已完成」状态
- [ ] 已完成项在 UI 上有明显区分（如划线、灰色）
- [ ] 状态变更后持久化，重启应用仍保留

**优先级**：高 | **估算**：1.5 小时 | **分配给**：—

---

## 4. 功能需求清单（简要）

| 编号 | 功能 | 优先级 |
|------|------|--------|
| F1 | 添加待办 | 高 |
| F2 | 编辑/删除待办 | 高 |
| F3 | 标记完成/未完成 | 高 |
| F4 | 列表持久化 | 高 |
| F5 | 按状态筛选（可选） | 中 |

---

## 5. 非功能需求（示例）

- **性能**：列表加载在 1 秒内完成。
- **可用性**：支持离线使用，数据存于本地。
- **兼容**：支持 Android / iOS 或 Web 其一即可（按课程要求）。

---

## 6. 与课程技术的对应

- 数据持久化：对应第 11 周 SharedPreferences 或文件。
- 列表与交互：对应第 9～10 周 ListView、表单与事件。
- 状态管理（可选）：对应第 13 周 Provider 等。
- 项目结构与 Git：对应第 14 周。

可将本示例扩展为小组的正式 PRD，并配合 Figma 原型与 GitHub Projects 使用。
