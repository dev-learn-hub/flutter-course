# 第15周课堂练习

本节为**当堂可完成的练习**，基于本周 [大纲](../outline/)、[讲义](../lecture/) 与 [演示案例](https://github.com/dev-learn-hub/flutter-course/tree/main/demos/week-15) 内容。

---

## 练习 1：阅读第15周案例并写出 1 条用户故事

**依据**：讲义「二、产品设计与需求分析」；案例 `demos/week-15/demo/prd_example.md`。

**步骤**：

1. 打开 `demos/week-15/demo/README.md` 与 `prd_example.md`，了解 PRD 结构与用户故事格式。
2. 任选一个你熟悉的小应用（如「记一笔账」「打卡签到」），按「作为…我想要…以便…」写 1 条用户故事。
3. 为该故事补充 2～3 条接受标准（可勾选形式）。

**验收**：能写出格式正确的一条用户故事及接受标准，并能说明其对应的「功能需求」。

---

## 练习 2：为同一功能做任务分解（WBS）

**依据**：讲义「四、小组协作与分工」；大纲「任务分解与工时估算」。

**步骤**：

1. 针对练习 1 中的用户故事，拆成 3～5 个子任务（如：UI 页面、数据保存、状态更新、简单测试）。
2. 为每个子任务写一句话描述，并估算工时（如 0.5h、1h、2h）。
3. （可选）在小组仓库的 GitHub Projects 中新建一个 Issue 或 Project 项，把上述任务录入。

**验收**：能列出可分配的子任务及合理估算，理解 WBS 与 Sprint 任务的关系。

---

## 练习 3：简述 Scrum 站会与本周作业的对应

**依据**：讲义「四、小组协作与分工」；大纲「Daily Standup」「Sprint 总结」。

**步骤**：

1. 用 2～3 句话说明：Daily Standup 通常要回答哪三个问题？
2. 说明本周作业中「2 周 Sprint」与「完成一功能模块、代码审查、Sprint 总结」如何对应到 Scrum 的 Planning / 开发 / Review / Retrospective。

**验收**：能正确说出站会三问，并能把作业要求对应到 Scrum 环节。
