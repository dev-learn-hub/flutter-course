# 每周内容制作方案模板

> 本文档是**方案模板**：规定每周内容制作的统一流程与产出结构。  
> 针对**某一周**制作具体方案时，请基于本模板生成该周的「本周具体方案」文档（如 `planning/week-04-plan.md`），并按要求填写各步骤的产出。

---

## 一、模板用途与使用方式

### 1.1 用途

- **统一流程**：每一周都按相同的 8 步执行，保证质量与可复现。
- **产出一致**：每周都产出「知识点体系大纲、案例、讲义、课堂练习、网站链接」，便于学生按同一入口（周聚合页）学习。
- **可追溯**：每步有明确产出物与检查项，便于自查或协作。

### 1.2 如何使用本模板制作「本周具体方案」

1. **复制「本周具体方案」结构**（见本文档第七节）到新文件，例如：  
   - 第 4 周：`planning/week-04-plan.md`  
   - 第 5 周：`planning/week-05-plan.md`  
   - 以此类推。
2. **替换占位**：将 `[第N周]`、`[本周主题]`、`[核心内容一句话]` 等替换为当周实际信息。
3. **按步骤执行**：从步骤 1 到步骤 8 依次进行，在「本周具体方案」中填写或链接每步的产出。
4. **勾选检查项**：完成一步即勾选该步的检查清单，便于跟踪进度。

---

## 二、全流程八步（标准步骤）

以下为每周内容制作的**标准步骤**，顺序不可颠倒；每步的详细说明与产出见第三节。

| 步骤 | 名称 | 核心产出 |
|------|------|----------|
| 1 | 看全学期课程大纲 | 对本周在全学期中的位置的简要记录 |
| 2 | 看本周课程大纲 | 本周目标、讲授要点、实操与作业的提取 |
| 3 | 看前面几周已有内容 | 前置依赖列表 + 可复用的表述/结构说明 |
| 4 | 生成本周知识点体系大纲 | `planning/week-XX-knowledge-map.md` |
| 5 | 上网搜索知识点，准备素材 | 素材清单（链接 + 用途 + 查阅日期/版本） |
| 6 | 编写本周案例，确保可运行 | `demos/week-XX/demo/`（及可选 `exercise/`） |
| 7 | 结合知识点与案例编写讲义与课堂练习 | `week-XX-lecture.md`、`week-XX-exercise.md`（课堂练习单独文件） |
| 8 | 组合内容、建立超链接、发布网站 | 周聚合页 / 大纲 / 讲义 / 案例 / 课堂练习 链接可访问且正确 |

## 三、各步骤说明与检查清单

### 步骤 1：先看全学期课程大纲

- **动作**：阅读 `index.md` 中的课程目标、周次安排表、成绩评定等。
- **目的**：明确本周在 14 周中的位置，与前后周的衔接关系，不超纲、不遗漏。
- **产出**（写入本周具体方案）：  
  - 本周是第几周；前一周/后一周主题；本周在「Dart 基础 / Flutter 基础 / 进阶 / 项目实践」中属于哪一阶段。

**检查清单**

- [ ] 已阅读全学期课程大纲（index.md）
- [ ] 已记录本周与前后周的主题衔接

---

### 步骤 2：看本周的课程大纲

- **动作**：阅读 `week-XX.md`（本周大纲），提取：周学习目标、讲授要点、实操任务、课后作业、学习检查清单。
- **目的**：以「本周大纲」为边界，后续知识点体系与案例、讲义均不超出该范围。
- **产出**（写入本周具体方案）：  
  - 本周学习目标（可复制或概括）；  
  - 讲授要点列表；  
  - 实操任务列表；  
  - 必做/选做作业。

**检查清单**

- [ ] 已阅读本周 `week-XX.md`
- [ ] 已提取目标、讲授要点、实操、作业

---

### 步骤 3：看前面几周已有内容

- **动作**：阅读前几周的 `week-XX.md`、若存在则阅读 `week-XX-lecture.md` 与 `demos/week-XX/` 的说明或代码。
- **目的**：明确「本周依赖前几周的哪些知识点」；复用表述风格与结构，保持学生体验一致。
- **产出**（写入本周具体方案）：  
  - **前置依赖**：列出前几周中与本周直接相关的知识点或能力（如「第 4 周：变量与函数；第 5 周：类与继承」）；  
  - **可复用**：前几周中可借鉴的表述、小节结构、案例命名方式等（一两句话即可）。

**检查清单**

- [ ] 已阅读前几周大纲（及已有讲义/案例）
- [ ] 已列出本周前置依赖
- [ ] 已注明可复用的表述或结构

---

### 步骤 4：生成本周的知识点体系大纲

- **动作**：在步骤 1、2、3 的基础上，把「本周课程大纲」拆解为可教、可练、可检验的知识点层级（建议 3～5 个主块，每块下若干小点）；每个知识点尽量带一句「学生应能做什么」。
- **目的**：形成本周的「施工图」，步骤 5（素材）、6（案例）、7（讲义）都对照该图执行。
- **产出**：  
  - 独立文件 `planning/week-XX-knowledge-map.md`，结构建议：
    - 本周主题与目标（一句话）
    - 知识点层级（一、二、三级标题或列表）
    - 每个知识点：名称 + 简要说明 + 「学生应能…」
    - （可选）与本周 demo / exercise 的对应关系

**检查清单**

- [ ] 已创建 `planning/week-XX-knowledge-map.md`
- [ ] 知识点不超出本周大纲范围
- [ ] 每个知识点有「学生应能做什么」或等价描述

---

### 步骤 5：上网搜索相关知识点的知识，准备素材

- **动作**：针对步骤 4 中的知识点，搜索官方文档、教程、常见问题与示例代码；记录来源与版本/日期。
- **目的**：保证讲义与案例注释有据可依，且便于日后按版本更新。
- **产出**（写入本周具体方案或单独素材清单）：  
  - **素材清单**：每条包含「主题 / 链接 / 用途（讲义用 / 案例注释用 / 常见问题用）/ 查阅日期或版本」。

**建议优先级**

- 优先：Flutter / Dart 官方文档、当前稳定版 release notes。  
- 其次：常见 Q&A（Stack Overflow、中文社区）、权威教程。  
- 对 API 或行为随版本变化的内容，标注查阅日期或 Flutter/Dart 版本。

**检查清单**

- [ ] 已为本周主要知识点搜索并保存素材
- [ ] 素材清单含链接与用途（及必要时版本/日期）

---

### 步骤 6：编写本周案例，确保案例能正确运行

- **动作**：在 `demos/week-XX/demo/` 下创建至少一个完整可运行的 Flutter 项目，对应本周核心知识点；可选在 `demos/week-XX/exercise/` 下放置练习用半成品。
- **目的**：先有可运行案例，再围绕案例写讲义，避免讲义与实现脱节。
- **产出**：  
  - `demos/week-XX/demo/<项目名>/`：可运行，关键处有注释；  
  - （可选）`demos/week-XX/exercise/`：题目说明 + 待补全位置；  
  - 在本地与 CI（若有）验证：`flutter pub get`、`flutter run` 等通过。

**检查清单**

- [ ] 已创建 `demos/week-XX/demo/` 且至少一个项目可运行
- [ ] 案例对应步骤 4 的知识点，不超纲
- [ ] （若做练习）已创建 `exercise/` 并写明要求
- [ ] 已在本地验证运行通过

---

### 步骤 7：结合知识点和案例，编写讲义与课堂练习

- **动作**：以步骤 4 的知识点体系为纲，以步骤 5 的素材与步骤 6 的案例为据，编写「操作导向」的讲义；分步、分平台（Windows/macOS）若有必要；常见问题单独小节。**另编写本周课堂练习**：单独文件，基于本周大纲、讲义与案例，列出当堂可完成的练习任务与验收要求。
- **目的**：学生能按讲义跟做，并能在讲义中看到对知识点与案例的引用；课堂练习提供明确的当堂任务与验收标准，与课后作业区分开。
- **产出**：  
  - 独立讲义：`week-XX-lecture.md`，permalink `/week-XX/lecture/`（推荐，与第 3 周一致）；  
  - 或：将「详细操作」合并进 `week-XX.md` 并在周聚合页说明（若本周不做独立讲义）。  
  - 讲义内须包含：本讲目标、与大纲的链接、与案例的链接（如「运行本课程 Demo」指向 `demos/week-XX/`）、练习/作业要求。  
  - **课堂练习**：独立文件 `week-XX-exercise.md`，permalink `/week-XX/exercise/`；内容基于本周大纲、讲义与案例，列出 2～3 个当堂练习（步骤、依据、验收），并链回大纲、讲义与周聚合页。

**检查清单**

- [ ] 讲义结构包含：目标、分步操作、常见问题、案例与练习说明
- [ ] 讲义中引用知识点或案例路径（如 `demos/week-XX/demo/...`）
- [ ] 若为独立文件，已设 `permalink: /week-XX/lecture/`
- [ ] 已创建 `week-XX-exercise.md`，含当堂练习任务与验收，已设 `permalink: /week-XX/exercise/`

---

### 步骤 8：组合内容，建立超链接，发布网站

- **动作**：  
  1. **将课堂练习加入周聚合页**：在 `week-XX-page.md` 的「本章资源」表中添加「课堂练习」一行，链接到 `./exercise/`（即 `/week-XX/exercise/`），与大纲、讲义、案例并列；表头可写「大纲 · 讲义 · 案例 · 课堂练习」。参考第 3 周：`week-03-page.md`。  
  2. 确认周聚合页中「大纲 / 讲义 / 案例 / 课堂练习」四处链接正确；检查讲义、课堂练习与大纲之间的互相链接。  
  3. **用 gh 或 git 推送并发布**：将本周所有变更提交后，执行 `git push origin main`（或使用 `gh` 等工具推送），触发仓库的 GitHub Pages 构建与部署；在 GitHub Actions 中确认 Pages 部署工作流执行成功。  
  4. **实际访问 GitHub Pages 验证**：在浏览器中打开课程站点的实际发布地址（如 `https://<org>.github.io/flutter-course/`），从课程首页进入本周聚合页，对「大纲、讲义、案例、课堂练习」四个链接逐一点击访问，确认无 404、页面内容与预期一致。
- **目的**：学生在网站上从「第 X 周」入口能顺畅到达大纲、讲义、案例、课堂练习，且无 404。
- **产出**：  
  - 周聚合页已正确链接大纲（`/week-XX/outline/`）、讲义（`/week-XX/lecture/` 或 `./outline/`）、案例（GitHub `demos/week-XX`）、**课堂练习（`/week-XX/exercise/`）**；  
  - 推送后站点可访问，链接有效。

**检查清单**

- [ ] 周聚合页「本章资源」表中四个链接正确（大纲、讲义、案例、课堂练习）
- [ ] 讲义与课堂练习内指向大纲、案例的链接正确
- [ ] 已用 gh 或 git 推送到 main，GitHub Pages 部署工作流成功
- [ ] 已实际访问发布的 GitHub Pages 网址，逐链验证聚合页 → 大纲 / 讲义 / 案例 / 课堂练习 均可达且内容正确
