# 第15周知识点体系大纲

> 本周主题与目标（一句话）：掌握产品需求分析（PRD、用户故事）、原型设计层次与工具、小组协作与 Scrum、任务分解与计划，能为小组项目撰写需求、设计原型并制定可执行计划。

---

## 一、产品设计与需求分析

| 知识点 | 简要说明 | 学生应能… |
|--------|----------|-----------|
| 产品开发流程 | 需求分析 → 产品设计 → 原型 → 开发 → 测试 → 发布迭代 | 说出各阶段及其产出 |
| PRD 文档 | 产品概述、核心功能、用户故事、功能/非功能需求、用户流程、成功指标 | 撰写或补全简易 PRD |
| 用户故事 | 作为…我想要…以便…；接受标准、优先级、估算 | 写出 2～3 条可验收的用户故事 |
| 功能/非功能需求 | 功能列表、性能/安全/可用性等约束 | 区分并列出核心功能与非功能需求 |

---

## 二、产品原型设计

| 知识点 | 简要说明 | 学生应能… |
|--------|----------|-----------|
| 低保真 / 中保真 / 高保真 | Wireframe、Mock-up、高保真 UI 的用途与区别 | 按阶段选择合适保真度 |
| 原型工具 | Figma、XD、Sketch、Miro | 能用至少一种工具做简单页面与流程 |
| Figma 基础 | 创建文件、形状/文本、组件、交互、分享 | 完成主要页面流程并标注或分享 |

---

## 三、小组协作与分工

| 知识点 | 简要说明 | 学生应能… |
|--------|----------|-----------|
| 角色分工 | PM、设计师、前端、后端、QA 的职责 | 在小组中认领或轮换角色 |
| Scrum 流程 | Sprint、Planning、Daily Standup、Review、Retrospective | 参与或主持站会与回顾 |
| 任务分解 WBS | 功能拆解为子任务、工时估算 | 将一功能拆成可分配任务并估算 |
| GitHub Projects | 看板、列表、任务与里程碑 | 用 Projects 管理本周/本组任务 |

---

## 四、版本与发布

| 知识点 | 简要说明 | 学生应能… |
|--------|----------|-----------|
| 版本号与 CHANGELOG | 语义化版本、Added/Changed/Fixed 等 | 维护简易 CHANGELOG 与发布检查清单 |

---

## 五、常见问题与防范

| 知识点 | 简要说明 | 学生应能… |
|--------|----------|-----------|
| 常见问题 | 进度延迟、技术风险、集成困难、沟通问题 | 识别并对应到防范策略 |
| 防范策略 | 风险评估、定期监控、团队建设与文档 | 在计划中纳入简单风险与沟通约定 |
