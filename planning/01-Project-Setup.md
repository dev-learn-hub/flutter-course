# 项目方案：Flutter 课程案例开发

## 整体目标

为《移动应用软件开发实训》课程开发配套教学案例，让学生可以：
- 拿到案例就能直接在本地运行
- 通过演示案例理解知识点
- 通过习题案例自己动手练习巩固
- 跟着讲义步骤一步步操作，环境能搭好，代码能跑通

## 目录结构设计

```
flutter-course/
├── index.md                # 课程大纲（已有）
├── week-XX.md              # 每周讲义（已有）→ 包含：
│   ├── 知识点讲解
│   ├── 详细操作指导（分Windows/macOS）
│   ├── 常见问题解决
│   └── 练习要求
├── demos/                  # 所有教学案例
│   ├── week-03/             # 按周次分组
│   │   ├── demo/            # 老师演示案例（完整可运行）
│   │   │   └── ...（完整项目文件）
│   │   └── exercise/        # 学生练习案例（半成品，需要学生补全代码）
│   │       └── ...（项目框架+待完成占位符）
│   ├── week-04/
│   └── ...
├── slides/                 # 教学PPT（后续补充）
│   └── week-XX.pptx
└── planning/               # 项目方案设计讨论记录
    └── 01-Project-Setup.md（本文档）
```

## 案例分类

### 1. 演示案例（demo）
- **完整可运行**：老师上课直接打开就能运行演示
- **知识点聚焦**：只展示本周知识点，不混合后面内容
- **代码带注释**：关键步骤注释说明，帮助学生理解

### 2. 练习案例（exercise）
- **半成品项目**：核心框架已经写好，留出关键位置让学生自己实现
- **配套说明**：在本周讲义里写明练习目标和要求
- **参考答案**：可以在 `exercise/solution` 放完整答案，方便学生对照

## 开发流程（按周推进）

从第3周开始，按顺序每周处理：

1. **理解知识点**：阅读本周 `week-XX.md` 讲义内容，明确本周要讲哪些知识点
2. **搜索资料**：确认环境安装步骤、知识点正确用法，保证内容准确
3. **创建案例结构**：在 `demos/week-XX/` 下创建 `demo/` 和 `exercise/`
4. **编写案例代码**：
   - 演示案例：完整实现，带注释
   - 练习案例：框架保留，删除关键代码留出空位
5. **本地验证运行**：确保案例能正常编译运行，步骤走得通
6 **完善讲义**：
   - 补全详细操作步骤
   - 分Windows/macOS分别说明
   - 添加常见问题和解决方法
7. **提交Git**：完成一周，继续下一周

## 质量要求

- **可运行**：所有案例必须本地实际运行通过，不能写了代码跑不起来
- **步骤准确**：环境安装步骤必须验证可行，记录遇到的坑和解决方法
- **知识点匹配**：案例完全匹配本周讲义内容，不超纲也不遗漏
- **简洁清晰**：案例聚焦知识点，不要多余复杂代码，方便学生理解
